# 《时空故事：青岚镇》MVP 多结局设计

## 1. 结局系统原则

《青岚镇》的结局不是由单一选项决定，而是由三十天内积累的世界状态、NPC 命运、关键事实和玩家行为共同判定。

结局系统遵循以下原则：

1. 结局必须由结构化条件触发。
2. 结局文本可以由 AI 辅助生成，但结局判定不能交给 AI。
3. 每个结局必须展示关键因果链。
4. 同一结局可以根据 NPC 生死、变量高低、玩家身份产生变体文本。
5. 灾难性结局优先级高于普通稳定结局。

## 2. 结局判定顺序

第 30 天夜晚结束后，按以下优先级检查：

```text
1. 特殊阵营 / 暗约结局
2. 灾难结局
3. 真相结局
4. 稳定结局
5. 个人声望类结局
6. 普通离开结局
```

如果多个结局同时满足，选择 priority 最高的结局，并在结局文本中追加其他已满足条件的“副结局评价”。

## 3. 结局一：青岚得存

```yaml
id: ending_qinglan_saved
name: 青岚得存
priority: 80
conditions:
  all:
    - fact: variables.plague_level
      less_than_or_equal: 40
    - fact: variables.bandit_power
      less_than_or_equal: 50
    - fact: variables.town_order
      greater_than_or_equal: 50
```

### 3.1 结局说明

青岚镇虽然经历疫病、粮价、匪患与旧案风波，但最终没有崩溃。药铺重新开门，镇门外再有商旅经过，集市虽不复往日热闹，却仍有人讨价还价。

### 3.2 主要达成方式

玩家通常需要做到：

- 控制疫病。
- 降低盗匪势力。
- 保持镇中秩序。
- 避免关键 NPC 大量死亡或失踪。

### 3.3 关键因果链示例

```text
你在第一批病人出现时协助药铺。
你及时补充药材，阻止沈青禾独自冒险。
商路遇袭后，你协助陆怀山追查盗匪。
集市粮价上涨时，你没有让冲突扩大。
最终，青岚镇保住了最艰难的三十天。
```

### 3.4 变体文本

| 条件 | 变体 |
|---|---|
| 沈青禾存活且 trust 高 | 药铺结尾会提到沈青禾继续行医，并感谢玩家。 |
| 陆怀山存活且 trust 高 | 镇门结尾会提到巡守恢复山路秩序。 |
| 罗守仓被说服开仓 | 集市结尾会提到米铺低价放粮。 |
| truth_progress >= 60 | 结尾会暗示旧案尚未完全结束。 |

## 4. 结局二：空镇余烬

```yaml
id: ending_empty_town
name: 空镇余烬
priority: 95
conditions:
  all:
    - fact: variables.plague_level
      greater_than_or_equal: 75
    - fact: variables.town_fear
      greater_than_or_equal: 70
```

### 4.1 结局说明

青岚镇没有在一夜之间被毁灭，而是在一天天的病痛、恐惧和逃离中空了。药铺门板紧闭，民居区挂满旧药包，集市只剩被风吹动的破布招牌。

### 4.2 常见触发原因

- 玩家长期忽视疫病线。
- 药材库存过低。
- 沈青禾失踪或死亡。
- 民居区病户无人救助。
- town_fear 长期过高。

### 4.3 关键因果链示例

```text
第 3 天，第一批病人来到药铺。
你没有及时介入，疫病开始扩散。
药材短缺后，沈青禾独自上山采药。
她未能及时返回，药铺治疗能力下降。
民居区病户越来越多，镇民开始逃离。
第 30 天，青岚镇成为空镇。
```

### 4.4 变体文本

| 条件 | 变体 |
|---|---|
| 沈青禾失踪 | 结尾强调药铺失去支撑。 |
| 老药师死亡 | 结尾强调回春药铺再无人开门。 |
| 小宛死亡或失踪 | 结尾提到流民棚空无一人。 |
| 玩家为医者 | 结尾会带有更强的失败感：医者未能救下此镇。 |

## 5. 结局三：匪火入镇

```yaml
id: ending_bandit_takeover
name: 匪火入镇
priority: 100
conditions:
  all:
    - fact: variables.bandit_power
      greater_than_or_equal: 80
    - fact: variables.guard_power
      less_than_or_equal: 40
```

### 5.1 结局说明

黑岭盗趁夜入镇，巡守无力抵挡。镇门火光冲天，祠堂牌位被烟熏黑，集市粮铺被洗劫一空。青岚镇没有迎来审判旧案的那一天，只迎来了刀火。

### 5.2 常见触发原因

- 玩家忽视商路遇袭。
- 没有协助陆怀山调查内应。
- guard_power 被集市冲突、夜袭、小规模战斗消耗。
- bandit_power 持续上升。
- 韩烬未被削弱或谈判牵制。

### 5.3 关键因果链示例

```text
商路被劫后，黑岭盗掌握了更多药材和粮食。
你没有追查山路踪迹，盗匪势力继续扩大。
集市冲突牵制了巡守力量。
陆怀山调查内应失败，镇门防备空虚。
第 30 天前夜，韩烬带人入镇。
```

### 5.4 变体文本

| 条件 | 变体 |
|---|---|
| 陆怀山死亡 | 结尾强调镇门无人指挥。 |
| 玩家为武人 | 结尾会描述玩家是否参与最后抵抗。 |
| 韩烬与玩家有暗约 | 可能不进入本结局，而进入“暗约成局”。 |
| 旧庙已发现 | 结尾提到韩烬夺走暗室中的证据。 |

## 6. 结局四：真相昭雪

```yaml
id: ending_truth_revealed
name: 真相昭雪
priority: 110
conditions:
  all:
    - fact: variables.truth_progress
      greater_than_or_equal: 80
    - fact: npc.npc_lu_huaishan.state.alive
      equals: true
    - fact: facts.old_case_record_found
      equals: true
```

### 6.1 结局说明

旧庙火案的真相终于被揭开。魏长庚多年隐藏的证词重见天日，青岚镇失去了一位镇长，却找回了迟来的公道。真相未必能治好所有创伤，但至少不再被埋在祠堂旧档之后。

### 6.2 常见达成方式

- 在酒肆获得旧案传闻。
- 打开祠堂旧档房。
- 与陆怀山或许闻舟合作调查。
- 发现破驿旧庙。
- 找到暗室证据。

### 6.3 关键因果链示例

```text
你在酒肆听到旧庙火案的传闻。
你说服老祠祝打开旧档房。
魏长庚试图封锁记录，却引起陆怀山警觉。
你追踪线索来到破驿旧庙。
暗室中的证据让旧案无法再被压下。
```

### 6.4 变体文本

| 条件 | 变体 |
|---|---|
| 魏长庚存活 | 可出现公开审问或退位。 |
| 魏长庚死亡 | 真相残缺，镇民争议更大。 |
| 许闻舟存活 | 结尾补充许闻舟完成手稿。 |
| plague_level 高 | 真相昭雪但镇中仍因疫病凋敝。 |
| bandit_power 高 | 真相揭开，但黑岭盗仍威胁小镇。 |

## 7. 结局五：暗约成局

```yaml
id: ending_secret_pact
name: 暗约成局
priority: 120
conditions:
  any:
    - all:
        - fact: facts.player_made_deal_with_han_jin
          equals: true
        - fact: variables.bandit_power
          greater_than_or_equal: 60
    - all:
        - fact: facts.player_helped_wei_suppress_truth
          equals: true
        - fact: variables.truth_progress
          less_than: 80
```

### 7.1 结局说明

青岚镇没有毁灭，却也没有真正得救。某些真相被继续压下，某些人被牺牲，某些交易在夜色中完成。玩家成为小镇新秩序的一部分，但这份秩序并不干净。

### 7.2 常见达成方式

- 与韩烬交易，换取小镇短期安全。
- 帮魏长庚压制旧案，换取权力或资源。
- 与赵三娘、罗守仓等人形成利益联盟。
- 放弃揭露某些秘密。

### 7.3 关键因果链示例

```text
你没有将赵三娘提供的情报告诉陆怀山。
你用旧案线索与魏长庚交换资源。
黑岭盗没有立刻入镇，因为你与韩烬达成了交易。
青岚镇保住了表面秩序，但真相仍被锁在暗处。
```

### 7.4 变体文本

| 条件 | 变体 |
|---|---|
| 与韩烬结盟 | 结尾偏黑暗，镇外山路由黑岭盗暗中控制。 |
| 帮魏长庚压制真相 | 结尾偏政治妥协，旧档被重新封存。 |
| 与罗守仓合作 | 结尾提到米铺与玩家形成资源联盟。 |
| 玩家为行商 | 结尾强调玩家带着利益离开。 |

## 8. 结局六：仁名远播

```yaml
id: ending_benevolent_name
name: 仁名远播
priority: 85
conditions:
  all:
    - fact: player.state.reputation
      greater_than_or_equal: 80
    - fact: variables.town_fear
      less_than_or_equal: 50
    - fact: facts.player_helped_refugees
      equals: true
```

### 8.1 结局说明

玩家未必解开所有秘密，也未必击败所有敌人，但许多人记住了玩家伸出的手。流民、病户、药铺、普通镇民都在不同地方留下了关于玩家的传言。

### 8.2 注意

该结局通常作为副评价存在，可以与“青岚得存”共同出现。如果灾难结局条件满足，则不会单独覆盖灾难结局。

### 8.3 关键因果链示例

```text
你救助了第一批病人。
你多次在民居区探访病户。
你没有驱逐流民，而是分出粮食和药材。
即使青岚镇仍有未解的旧事，镇民记住了你的仁名。
```

## 9. 结局七：血债难消

```yaml
id: ending_blood_debt
name: 血债难消
priority: 90
conditions:
  all:
    - fact: player.stats.kill_count
      greater_than_or_equal: 8
    - fact: variables.town_order
      less_than_or_equal: 45
```

### 9.1 结局说明

许多麻烦被你用刀解决，但刀不能缝合人心。黑岭盗或许退了，米铺或许开仓了，旧案或许被逼出水面，可镇中人看你的眼神也变了。

### 9.2 触发方式

- 频繁使用战斗、威胁、强夺。
- 杀死或逼死多个 NPC。
- 用暴力解决集市、流民、盗匪、旧案冲突。
- town_order 没有因暴力恢复，反而下降。

### 9.3 变体文本

| 条件 | 变体 |
|---|---|
| 玩家为武人 | 结尾强调“武人仍在路上”。 |
| 陆怀山敌对 | 巡守会将玩家列为危险人物。 |
| 沈青禾低信任 | 药铺不会为玩家送行。 |

## 10. 结局八：孤身远行

```yaml
id: ending_lone_departure
name: 孤身远行
priority: 10
conditions:
  all:
    - fact: player.state.alive
      equals: true
```

### 10.1 结局说明

三十天过去，青岚镇仍在风雨里。你没有彻底救下它，也没有彻底毁掉它。你带着一些人的感谢、一些人的怨恨和更多未解之事，离开了镇门。

### 10.2 触发方式

当没有满足其他高优先级结局，但玩家存活时，进入该结局。

### 10.3 变体文本

| 条件 | 变体 |
|---|---|
| 沈青禾 trust 高 | 她会送玩家一包药。 |
| 陆怀山 trust 高 | 他会在镇门送行。 |
| 赵三娘 trust 高 | 她会留下一句半真半假的提醒。 |
| 小宛获救 | 小宛会目送玩家离开。 |

## 11. NPC 命运总结

每个结局页面都应展示核心 NPC 命运。

示例：

```yaml
npc_fate_summary:
  npc_shen_qinghe:
    alive_true: 沈青禾继续守着回春药铺。
    missing_true: 沈青禾再也没有从山林回来。
    dead_true: 回春药铺少了最年轻的药师。

  npc_lu_huaishan:
    alive_true: 陆怀山继续追查镇中余患。
    dead_true: 镇门巡守失去了最可靠的人。

  npc_wei_changgeng:
    truth_revealed: 魏长庚终于为旧案付出代价。
    truth_hidden: 魏长庚仍坐在祠堂前，像什么都没有发生。
```

## 12. 结局页面结构

结局页面应包含：

```text
结局名称
结局主文案
关键世界变量
关键因果链
核心 NPC 命运
玩家称号
副评价
重新开始 / 查看事件日志
```

## 13. 玩家称号

根据玩家行为追加称号：

| 称号 | 条件 |
|---|---|
| 青岚义士 | reputation 高，救助多名 NPC |
| 山路斩匪人 | bandit_power 大幅降低，战斗贡献高 |
| 回春同行者 | 疫病线贡献高，沈青禾 trust 高 |
| 旧案见证者 | truth_progress 高 |
| 冷眼旁观者 | 行动少，关键事件多次忽视 |
| 暗局行商 | 与多方交易，隐藏事实较多 |
| 血债客 | kill_count 高 |

## 14. 结局设计原则

1. 结局不是奖励或惩罚，而是因果总结。
2. 灾难结局也要让玩家明白原因。
3. 好结局不应要求完美解决所有问题。
4. 黑暗结局也要有其合理性和代价。
5. 每个核心 NPC 的命运都应反映玩家与世界状态。
