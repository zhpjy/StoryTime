# 《时空故事：青岚镇》MVP 地图与地点设计

## 1. 地图设计目标

青岚镇 MVP 地图暂时不使用美术大地图，也不使用 Canvas 或 PixiJS。首版地图完全由前端绘制，采用棋盘式网格。

地图的作用不是单纯导航，而是作为“空间叙事入口”。玩家通过地图观察：

```text
哪里有 NPC。
哪里有建筑。
哪里有事件。
哪里危险。
哪里被封锁。
哪里出现新变化。
```

每个地块都可以根据时间、事件和世界变量改变显示状态。

## 2. 地图规格

```yaml
map:
  id: map_qinglan_town
  name: 青岚镇
  width: 8
  height: 6
  render_type: css_grid
```

地图为 8 列 × 6 行，共 48 个地块。MVP 中只有 8 个核心地点绑定完整交互，其余地块作为道路、山林、河岸、空地等过渡空间。

## 3. 前端交互要求

### 3.1 鼠标悬浮地块

悬浮时显示简要信息：

```text
地点名称
地形
危险程度
当前 NPC
当前建筑
当前事件
是否可进入
```

### 3.2 点击地块

点击后右侧显示地块详情面板：

```text
地点描述
当前时间段状态
在场 NPC
可进入建筑
当前事件
可执行事项
不可执行事项及原因
```

### 3.3 地块显示状态

| 状态 | 地图表现 |
|---|---|
| 当前选中 | 金色边框 |
| 鼠标悬浮 | 柔和高亮 |
| 未发现 | 暗色问号 |
| 有 NPC | 人物标记 |
| 有建筑 | 建筑标记 |

## 6.1 镇门

```yaml
id: loc_town_gate
name: 镇门
type: town_location
tags:
  - entrance
  - guard
  - refugee
  - merchant_route
state:
  status: normal
  is_accessible: true
  danger_level: 10
  refugee_count: 3
  guard_presence: 2
  merchant_queue: 1
```

### 地点定位

镇门是外部世界进入青岚镇的入口，也是商队、流民、巡守和盗匪消息最先出现的地方。

### 时间段表现

| 时间段 | 描述 |
|---|---|
| 晨 | 镇门初开，巡守检查来往行人。 |
| 午 | 商旅与流民聚在门外，气氛有些紧张。 |
| 夜 | 镇门半闭，火把下只剩巡守来回走动。 |

### 主要建筑

| 建筑 ID | 名称 | 功能 |
|---|---|---|
| building_guard_post | 巡守岗 | 打听盗匪、协助巡逻、接受委托 |
| building_town_notice | 告示牌 | 查看公告、悬赏、商路消息 |

### 常驻 / 相关 NPC

| NPC | 出现条件 |
|---|---|
| 陆怀山 | 多数时间在镇门或集市巡逻 |
| 流民小宛 | refugee_pressure 达到一定值后出现 |
| 商队管事 | merchant_route_safety 较高时出现 |

### 可操作事项

| 行动 | 条件 | 效果 |
|---|---|---|
| 打听山路消息 | 任意白天 | 获得盗匪、商队、流民传闻 |
| 协助检查流民 | refugee_pressure >= 20 | town_order +3，refugee_pressure 可下降或上升 |
| 协助巡逻 | 陆怀山在场 | guard_power +3，陆怀山 trust +5 |
| 离开小镇 | 特定剧情或结局前 | 可能进入“孤身远行”方向 |

### 状态变化

- 如果 `bandit_power >= 70`，镇门危险度上升。
- 如果 `guard_power <= 30`，镇门可能被封锁。
- 如果 `refugee_pressure >= 60`，镇门触发流民冲突事件。

## 6.2 集市

```yaml
id: loc_market

  status: normal

  conflict_level: 10
```

### 地点定位

集市是青岚镇最热闹的地方，也是粮价线、流民线、谣言线的核心。

### 时间段表现

| 时间段 | 描述 |
|---|---|
| 晨 | 摊贩陆续摆开货物，米铺前已有几人排队。 |
| 午 | 集市最热闹，讨价还价声此起彼伏。 |
| 夜 | 集市空了大半，只剩几个小贩收拾摊位。 |

### 主要建筑

| 建筑 ID | 名称 | 功能 |
|---|---|---|
| building_rice_shop | 罗家米铺 | 买粮、调查囤粮、粮价线 |
| building_blacksmith | 铁匠铺 | 修理工具、购买简易武器 |
| building_vendor_stalls | 摊贩区 | 打听消息、购买杂物 |

### 相关 NPC

| NPC | 作用 |
|---|---|
| 罗守仓 | 米铺掌柜，粮价线核心 |
| 铁生 | 铁匠，武人线辅助 NPC |
| 小宛 | 流民少女，流民线与粮价线连接点 |

### 可操作事项

| 行动 | 条件 | 效果 |
|---|---|---|
| 购买粮食 | 米铺开放且玩家有钱 | 获得食物，money 下降 |
| 打听集市传闻 | 白天或酒肆传闻联动 | 获得当前世界状态提示 |
| 调查米粮去向 | grain_shortage_rumor_happened = true | truth 或 food 线推进 |
| 安抚争执人群 | reputation 或 negotiation 足够 | town_fear -5，town_order +3 |
| 揭露囤粮 | 找到证据 | food_price -20，town_order 可能 -10 |

### 状态变化

- `food_price >= 130` 时，集市冲突增加。
- `town_fear >= 60` 时，传闻变多，部分商贩闭市。
- 罗守仓囤粮被揭露后，集市短期混乱，但长期粮价下降。

## 6.3 回春药铺

```yaml
id: loc_herb_shop
name: 回春药铺
type: town_location
tags:
  - medicine
  - treatment
  - plague
state:
  status: normal
  is_accessible: true
  danger_level: 5
  herb_stock: 45
  patient_count: 3
  is_quarantined: false
```

### 地点定位

回春药铺是疫病线核心地点。它体现时间压力最明显：病人会增加，药材会减少，NPC 会疲惫或离开。

### 时间段表现

| 时间段 | 描述 |
|---|---|
| 晨 | 药铺刚刚开门，药童正在整理药斗。 |
| 午 | 求诊的人渐渐多了，药香和咳声混在一起。 |
| 夜 | 前门半掩，后堂仍有灯火。若疫病严重，沈青禾会彻夜熬药。 |

### 主要建筑

| 建筑 ID | 名称 | 功能 |
|---|---|---|
| building_herb_counter | 药柜 | 买药、查库存、偷药 |
| building_consulting_room | 问诊间 | 治疗、帮忙诊治、打听疫病 |
| building_back_yard | 后院 | 调查药材来源、发现旧药筐线索 |

### 相关 NPC

| NPC | 作用 |
|---|---|
| 沈青禾 | 疫病线核心，药铺学徒 |
| 老药师 | 沈青禾师父，前期提供医学线索 |

### 可操作事项

| 行动 | 条件 | 效果 |
|---|---|---|
| 购买药材 | herb_stock > 0，白天 | 获得药材，money 下降 |
| 求医治疗 | health < 100，herb_stock >= 5 | health 上升，herb_stock 下降 |
| 帮忙熬药 | plague_level >= 30，stamina 足够 | plague_level -3，沈青禾 trust +5 |
| 打听疫病 | 沈青禾在场 | 获取病源线索 |
| 偷取药材 | 夜晚，药柜未锁 | 获得药材，但失败会降低关系和秩序 |
| 调查药材来源 | 后院解锁 | 发现山林或旧庙相关线索 |

### 状态变化

- `plague_level >= 40` 时，patient_count 增加。
- `herb_stock < 25` 且 plague_level 高时，沈青禾可能上山采药。
- 如果沈青禾失踪或死亡，药铺治疗能力大幅下降。
- 如果 plague_level >= 70，药铺可能被封锁或隔离。

|---|---|

| NPC | 出现条件 |
|---|---|
| 沈青禾 | 药材短缺后可能上山 |

| 行动 | 条件 | 效果 |

- `bandit_power` 高时，山林危险大幅上升。
- `herb_resource` 被过度采集后，采药收益下降。
- 沈青禾上山后，如果玩家不救，她可能受伤、失踪或被盗匪劫走。

## 6.8 破驿旧庙

```yaml
id: loc_old_temple
name: 破驿旧庙
type: ruin_location
tags:
  - hidden
  - final_clue
  - danger
  - old_case
state:
  status: hidden
  is_accessible: false
  danger_level: 50
  discovered: false
```

### 地点定位

破驿旧庙是旧案线和匪患线交汇的隐藏地点。它不是开局可见地点，需要通过传闻、追踪、旧档、山林探索等方式发现。

### 解锁方式

满足任一条件可发现旧庙：

```yaml
unlock_conditions:
  any:
    - fact: facts.old_case_record_found
      equals: true
    - fact: facts.player_tracked_bandit_to_old_temple
      equals: true
    - fact: facts.shen_hidden_herb_note_found
      equals: true
    - fact: relationship.player.npc_zhao_sanniang.trust
      greater_than_or_equal: 40
```

### 时间段表现

| 时间段 | 描述 |
|---|---|
| 晨 | 旧庙藏在荒草后，残破屋脊几乎被藤蔓遮住。 |
| 午 | 破败神像前积着厚灰，似乎很久无人来过。 |
| 夜 | 风声穿堂，像有人在低声说话。 |

### 主要建筑

| 建筑 ID | 名称 | 功能 |
|---|---|---|
| building_ruined_temple | 旧庙正殿 | 搜查旧案残迹 |

### 可操作事项

| 行动 | 条件 | 效果 |
|---|---|---|
| 搜查旧庙 | discovered = true | truth_progress +10 |
| 进入暗室 | 找到入口 | 触发最终线索事件 |
| 埋伏盗匪 | 陆怀山协助或武人身份 | 降低 bandit_power |
| 救出被囚 NPC | 特定 NPC 被劫后 | 改变 NPC 命运 |

### 状态变化

- 如果韩烬势力过高，旧庙会成为盗匪临时据点。
- 如果魏长庚压制真相，可能派人提前销毁证据。
- 如果玩家过晚发现旧庙，部分 NPC 命运可能已无法挽回。

## 7. 地点之间的空间关系

```text
镇门 ←→ 集市 ←→ 药铺
  ↓       ↓       ↓
酒肆 ←→ 祠堂 ←→ 民居区
  ↓       ↓       ↓
外路 ←→ 山林 ←→ 破驿旧庙
```

基础移动规则：

- 相邻地块移动消耗 1 行动点。
- 从镇内到山林消耗 1 到 2 行动点，视状态而定。
- 夜晚进入山林或旧庙会增加危险判定。
- 地点被封锁时不可进入，除非满足特殊条件。

## 9. 地点设计原则

1. 每个核心地点至少绑定 1 条主线或支线。
2. 每个地点至少有 2 个建筑或区域。
3. 每个地点至少有 3 种可操作事项。
4. 同一地点在晨、午、夜应有不同表现。
5. 地点状态必须能被事件改变。
6. 地点交互必须由 conditions 和 effects 驱动，不能只写描述文本。
