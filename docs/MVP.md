# 《时空故事：青岚镇》MVP 实现说明

本项目实现一个小型动态叙事模拟游戏：玩家在 30 天内通过地点行动、NPC 互动、事件处理影响青岚镇的疫病、盗匪、粮价、秩序、恐惧与旧案真相。

## 核心循环

```text
选择身份 → 点击地图 → 查看地点 → 执行动作 → 消耗行动点 → 推进时间 → NPC 自主行动 → 事件触发 → 世界状态变化 → 第 30 天结局判定
```

## MVP 完整模块

- 时间系统：30 天，晨、午、夜，每段 3 行动点。
- 地图系统：8 × 6 CSS Grid 棋盘地图。
- 地点系统：镇门、集市、回春药铺、旧酒肆、祠堂、民居区、青岚山林、破驿旧庙。
- 建筑系统：地点内二级交互对象。
- NPC 系统：核心 NPC、重要 NPC、普通 NPC。
- 行为系统：条件判断 + 效果执行。
- 事件系统：时间事件、地点事件、NPC 行为事件、世界状态事件、玩家选择事件。
- 对话系统：初见、闲谈、地点打听、人物打听、事件反应、高低关系、秘密揭露。
- 关系系统：信任、畏惧、感激、怀疑。
- 事实系统：用于记录关键剧情状态。
- 结局系统：结构化条件判定 + 因果链展示。
- Story Lab：内容浏览、Prompt 模板、草稿校验、模拟测试、导出。
- Validator：Schema、引用、路径、效果、逻辑与缺口分析。
